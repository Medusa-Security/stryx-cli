# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in STRYX itself, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, please use [GitHub Security Advisories](https://github.com/medusa-Security/stryx/security/advisories/new) to report vulnerabilities privately.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | Yes                |

## Response Time

- Initial acknowledgment: within 48 hours
- Status update: within 7 days
- Fix timeline: depends on severity, typically within 30 days for critical issues

## Scope

This security policy applies to vulnerabilities in STRYX itself (the DAST tool). It does not apply to vulnerabilities found in applications tested by STRYX.

## Authorized Testing Only

STRYX is a security testing tool intended for use on systems you own or have explicit written authorization to test. Running STRYX against systems without authorization is illegal and unethical.

## Disclosure Policy

We follow coordinated disclosure. Please do not publicly disclose vulnerabilities until a fix has been released.
